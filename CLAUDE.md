# CLAUDE.md

Guidance for Claude Code when working in this repository.

## What this is

**PotLuckHub** (display name **Potluck**) — a native **SwiftUI** iPhone app for the Potluck
home-cook / chef marketplace in Singapore. It's a thin client over a live REST backend; there is
no local database. iOS deployment target **17.0**, portrait-only, iPhone-only
(`TARGETED_DEVICE_FAMILY = 1`), runs on iPad in iPhone-compatibility mode.

## Project generation & build

The Xcode project is **generated by XcodeGen** from [project.yml](project.yml) — do **not** hand-edit
`Potluck.xcodeproj` (it's regenerated and will be overwritten).

```bash
xcodegen generate                 # regenerate Potluck.xcodeproj after changing project.yml or adding files
# Simulator build:
xcodebuild -project Potluck.xcodeproj -scheme Potluck -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro Max' build
```

Adding a new `.swift` file under `Potluck/` is picked up automatically on the next
`xcodegen generate` (sources are globbed from the `Potluck` folder).

## Architecture

```
Potluck/
  App/          PotluckApp (entry), RootView (TabView), Theme (colors/fonts)
  Auth/         AuthManager (@MainActor ObservableObject, session state), Keychain (token store)
  Networking/   APIClient (URLSession wrapper), PotluckService (endpoint calls), Models (Codable)
  Components/   SharedComponents (AvatarView, Pill, PrimaryButton, potluckCard, BrandMark, …)
  Features/     Explore, Dishes, Bookings, Booking (request), Profile (+ AuthSheet)
```

- **State**: `AuthManager` is the single source of truth for the signed-in user, injected as an
  `@EnvironmentObject`. Tokens are persisted in the Keychain and restored on launch.
- **Tabs**: Explore and Dishes are browsable **without** signing in; Bookings and Profile require auth.

## Backend API — important conventions

Base URL: **`https://api.potluckhub.io/api/v1`** (Fastify; backend repo lives separately at
`~/projects/tertiary/potluck`, deployed via Coolify/Docker). All calls go through
[APIClient.send](Potluck/Networking/APIClient.swift).

- **Response envelope**: every response is `{ success, data, error?, pagination? }`. `send()` unwraps
  `data` on success and throws `APIError.server(message)` using `error.message` otherwise.
- **Prices are integer cents** — format with `Money.format(_:currency:)`, never display raw.
- **Inconsistent number types**: ratings come back as a JSON **String** at chef level but a **Number**
  on menus. Decode such fields with `FlexNumber`, which accepts either.
- **Auth**: `auth/login`, `auth/register` (role `customer`), `auth/me`, **`DELETE auth/account`**
  (account deletion). Pass `authenticated: true` to attach the Bearer token.

## App Review rejection lessons (must respect before submitting)

- **2.3.3 Accurate Metadata** — App Store screenshots must be real captures of the *current* app's
  working screens (Explore / Dishes / Chef detail). Never use marketing mockups, Google Play assets,
  or splash/login screens; the majority must show core functionality.
- **5.1.1(v) Account deletion** — the app has login, so it **must** offer in-app account deletion
  (Profile → "Delete Account" → confirm → `DELETE auth/account` → sign out). Provide a screen
  recording of the flow (captured on a physical device) in the App Review notes.
- **2.1 Completeness** — verify the demo account exists/logs in and the build launches without
  crashing on iPad before submitting.
